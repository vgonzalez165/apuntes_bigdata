# PR0901. Procesamiento de Datos Distribuido con AWS EMR y Apache Spark

## 1. Objetivo

El objetivo de esta práctica es familiarizarse con el ecosistema de Big Data en la nube, aprendiendo a desplegar un clúster gestionado de **Amazon EMR (Elastic MapReduce)** y a ejecutar un job de **Apache Spark** para procesar un conjunto de datos.


## 3. Enunciado

### Fase 1: Preparación del Almacenamiento (S3)

1.  Crea un bucket de S3 con un nombre único (ej: `practica-emr-spark-[tu-nombre]`).
2.  Dentro del bucket, crea tres carpetas:
    * `/scripts`: donde subirás el código de Spark.
    * `/input`: donde colocarás los datos de entrada (puedes usar un archivo `.csv` o `.txt` pequeño, como un dataset de "WordCount").
    * `/output`: donde Spark guardará los resultados (esta carpeta debe estar vacía o no existir antes de ejecutar el programa).

### Fase 2: Desarrollo del programa Spark

Desarrolla un script sencillo en Python (**PySpark**) que realice las siguientes tareas:

- Leer un archivo de texto desde la ruta de S3 `input`.
- Contar la ocurrencia de cada palabra (WordCount) o realizar un filtrado/agregación simple.
- Escribir el resultado en formato Parquet o CSV en la ruta de S3 `output`.

### Fase 3: Lanzamiento del clúster EMR

Despliega un clúster con la siguiente configuración mínima:

* **Release:** emr-6.x (o superior).
* **Aplicaciones:** Spark, Hadoop, Zeppelin.
* **Instancias:** 1 Nodo Maestro y 2 Nodos Core (tipo `m5.xlarge` o `t3.medium` si estás en capa gratuita/pruebas).
* **Seguridad:** Selecciona tu par de claves EC2 para permitir acceso si es necesario.

### Fase 4: Ejecución del Job

Existen dos formas de ejecutarlo (elige una):
1.  **Modo Steps (Pasos):** Añade un "Step" al clúster de tipo "Spark application", apuntando al script en S3 y pasando los argumentos de las rutas de entrada y salida.
2.  **Modo Interactivo:** Accede vía SSH al nodo maestro y ejecuta el comando `spark-submit`.

### Fase 5: Validación y Limpieza

1.  Verifica en la consola de S3 que los archivos de salida se han generado correctamente.
2.  **IMPORTANTE:** Una vez finalizada la comprobación, termina (Terminate) el clúster de EMR para evitar costes adicionales y elimina los recursos de S3 si no son necesarios.
